Agro ML - tinyML model for identify tomato plant related diseases

>A quantized MobileNetV2 model trained to classify tomato plant diseases from images, converted to INT8 TFLite and packaged as a C header for direct use on ESP32 microcontrollers.

## 2. Dataset — PlantVillage

Trained on the [PlantVillage dataset](https://github.com/spMohanty/PlantVillage-Dataset), using the tomato leaf subset (colour images).

**10 classes:**

|Class|Label|
|---|---|
|Bacterial Spot|`Tomato_Bacterial_spot`|
|Early Blight|`Tomato_Early_blight`|
|Late Blight|`Tomato_Late_blight`|
|Leaf Mold|`Tomato_Leaf_Mold`|
|Septoria Leaf Spot|`Tomato_Septoria_leaf_spot`|
|Spider Mites|`Tomato_Spider_mites_Two_spotted_spider_mite`|
|Target Spot|`Tomato_Target_Spot`|
|Yellow Leaf Curl Virus|`Tomato_Tomato_Yellow_Leaf_Curl_Virus`|
|Mosaic Virus|`Tomato_Tomato_mosaic_virus`|
|Healthy|`Tomato_healthy`|

The dataset has notable class imbalance which was handled using per-class weights during training.

## 3. Model

**Base:** MobileNetV2 (`alpha=0.5`, pretrained on ImageNet, `include_top=False`)

**Input:** 96 × 96 × 3 (RGB)

**Classification head:**

Python

```
GlobalAveragePooling2D()
Dropout(0.2)
Dense(number_classes, activation='softmax')
```

MobileNetV2 was chosen because its depthwise separable convolutions and inverted residual blocks keep the parameter count and memory footprint low. `alpha=0.5` reduces the channel width, drastically shrinking the model size, and 96×96 is an aggressive but practical balance between image detail and inference speed for microcontrollers.

## 4. Training

The training process uses the Keras `image_dataset_from_directory` utility with `AUTOTUNE` prefetching for performance.

**Training Strategy:** The base MobileNetV2 model is frozen (`base_model.trainable = False`), and only the custom classification head is trained for 15 epochs using the Adam optimizer.

**Data augmentation** (applied via Keras sequential layers):

Python

```
keras.Sequential([
    keras.layers.RandomFlip('horizontal_and_vertical'),
    keras.layers.RandomRotation(0.2),
    keras.layers.RandomZoom(0.1),
])
```

**Callbacks:**

- `ModelCheckpoint`: Saves the best weights to `best_model.keras`.
    
- `EarlyStopping`: Halts training if `val_loss` stops improving for 4 epochs, restoring the best weights.

## 5. TFLite Conversion & INT8 Quantization

**Why INT8?** Converting from float32 to INT8 speeds up inference significantly on microcontrollers and drastically cuts model size, usually with an accuracy drop of under 1–2%.

**All-in-one Conversion Pipeline:** The `convertor.py` script handles the entire pipeline:

1. **Sanity Check:** Runs the original Keras model against the test data to ensure the base model is actually functional before conversion.
    
2. **Float32 Conversion:** Creates a baseline TFLite model.
    
3. **INT8 Quantization:** Uses a representative dataset generator to calibrate activation ranges and convert to full INT8.
    
4. **C Header Generation:** Automatically extracts the hex bytes of the INT8 model, alongside its scaling factors and zero-points, and writes them directly to `output/model.h`.

**Model sizes:**

| Format          | Size        |
| --------------- | ----------- |
| float32 Keras   | ~3.5 MB     |
| float32 TFLite  | ~2.7 MB     |
| **INT8 TFLite** | **~982 KB** |

## 6. Using the Model on ESP32

Copy `output/model.h` into your firmware project and include it. The model runs with the TensorFlow Lite for Microcontrollers library (e.g., `tanakamasayuki/TensorFlowLite_ESP32`).

**Basic setup:**

C++

```
#include "model.h"
#include <TensorFlowLite_ESP32.h>

constexpr int kTensorArenaSize = 1024 * 512;  // Adjust as needed
uint8_t tensor_arena[kTensorArenaSize];

tflite::MicroMutableOpResolver<10> resolver;
resolver.AddConv2D();
resolver.AddDepthwiseConv2D();
resolver.AddFullyConnected();
resolver.AddAveragePool2D();
resolver.AddSoftmax();
resolver.AddReshape();
resolver.AddAdd();

tflite::MicroInterpreter interpreter(
    tflite::GetModel(MODEL_DATA), resolver, tensor_arena, kTensorArenaSize, nullptr
);
interpreter.AllocateTensors();
```

**Inference:**

1. Resize your image to **96 × 96 RGB**.
    
2. Pass raw `[0, 255]` pixel values into the input tensor (the model has preprocessing baked in).
    
3. Call `interpreter.Invoke()`.
    
4. Read the output tensor, applying the output scale and zero-point defined in `model.h`.

## 7. Scripts
_Note: The scripts use hardcoded configuration variables at the top of each file (e.g., `IMG_SIZE`, `EPOCHS`, `DATASET_DIR`). Modify those variables directly in the code to change paths or parameters._

|**Script**|**What it does**|
|---|---|
|`train_model.py`|Trains the Keras model, generates plots, and exports `plant_disease_model.keras`.|
|`test_model.py`|Contains functions to evaluate the model via batch testing or single-image inference.|
|`convertor.py`|Validates Keras accuracy, creates Float32/INT8 TFLite models, and generates `model.h`.|

**Train:**

```
python train_model.py
```

**Evaluate:**

Edit `test_model.py` to call either `batch_data_testing()` or `single_data_testing('path/to/image.jpg')` at the bottom of the script, then run:

```
python test_model.py
```

**Convert:**

Ensure your test images are in `data/test` and run:

```
python convertor.py
```

## 8. Known Issues
#### 1. Dataset Bias ("Lab vs. Wild")
The model is trained on the PlantVillage dataset, which features leaves against uniform, clean backgrounds. Real-world deployment introduces complex natural noise (soil, shadows, and debris) that can degrade prediction accuracy.

#### 2. Lack of an "Unknown" Class
Because the classification head uses a standard Softmax activation function over 10 distinct tomato disease categories, the output probabilities must always sum to 100%. If a completely unrelated object or an unlearned leaf is presented, the model will still force-classify it into one of the 10 target classes with high confidence.

#### 3. Granularity Loss from Downsampling (96x96)
Resizing images down to 96×96 pixels discards critical high-frequency spatial details. Fine-grained visual symptoms—such as the tiny lesions characteristic of Septoria Leaf Spot or early-stage Blight—tend to blur together, resulting in a higher rate of misclassification between look-alike diseases.

#### 4. Reduced Feature Capacity
Using a lower width multiplier (alpha=0.5) cuts the number of operational filters across the MobileNetV2 architecture in half. While this reduction successfully optimizes memory footprint, it lowers the model's absolute capacity to extract highly nuanced abstract features, capping its top-end accuracy compared to a full-sized model.
#### 5. Memory Allocation Restraints
The runtime calculation space (Tensor Arena) demands substantial raw memory on-device. Even though the compressed weights fit comfortably, memory fragmentation on a microcontroller can trigger runtime allocation failures (kTfLiteError).
